📚 Что такое коллекция и итератор?

📦 Коллекция (контейнер)

Объект, который хранит группу других объектов и предоставляет методы для работы с ними: добавление, удаление, поиск, перебор.

MyContainer<Employee*> company;
company.push_back(new Developer("Alice", 101, 75000, "C++", 450));

➡️ Итератор

Объект, который указывает на элемент внутри коллекции и позволяет перемещаться по ней. Работает как «умный указатель».

for (auto it = company.begin(); it != company.end(); ++it) {
    (*it)->work();
}
💡 Преимущества использования итераторов:
• Скрывают внутреннее устройство коллекции (массив, список, дерево)
• Позволяют перебирать элементы единообразно
• Дают возможность использовать коллекцию в цикле for с диапазоном (range-based for)
• Позволяют использовать стандартные алгоритмы (std::find, std::sort)

🏗️ Иерархия классов предметной области

🔷 MyContainer<T> — шаблонная коллекция: может хранить объекты любого типа. В нашем случае хранит указатели на Employee (полиморфно).

💻 Реализация MyContainer<T>

Основные методы

template <typename T>
class MyContainer {
private:
    T* data;          // указатель на динамический массив
    size_t capacity;  // вместимость
    size_t count;     // текущее количество элементов

public:
    MyContainer();                           // конструктор
    ~MyContainer();                          // деструктор
    
    void push_back(const T& value);            // добавление в конец
    void pop_back();                           // удаление последнего
    T& operator[](size_t index);              // доступ по индексу
    size_t size() const;                      // размер
    bool empty() const;                       // проверка на пустоту
    void clear();                               // очистка
};

Внутренний итератор (с поддержкой random_access_iterator_tag)

class Iterator {
public:
    // Типы, необходимые для std::iterator_traits
    using iterator_category = std::random_access_iterator_tag;
    using value_type = T;
    using difference_type = std::ptrdiff_t;
    using pointer = T*;
    using reference = T&;
    
    // Операторы, поддерживаемые итератором
    reference operator*() const;
    Iterator& operator++();      // префиксный инкремент
    Iterator operator++(int);   // постфиксный инкремент
    Iterator& operator--();      // префиксный декремент
    bool operator==(const Iterator& other) const;
    bool operator!=(const Iterator& other) const;
};
✅ Итератор поддерживает все операции категории random_access_iterator_tag: чтение/запись, движение вперёд/назад, арифметические операции (it + 5, it += 2), сравнение, индексирование (it[3]).

🔍 Использование std::find и std::sort

// Поиск сотрудника по ID с использованием std::find_if
auto it = std::find_if(container.begin(), container.end(), 
    [id](Employee* emp) { return emp->getId() == id; });

// Сортировка сотрудников по зарплате
std::sort(container.begin(), container.end(), 
    [](Employee* a, Employee* b) { return a->getSalary() < b->getSalary(); });

// Сортировка сотрудников по ID
std::sort(container.begin(), container.end(), 
    [](Employee* a, Employee* b) { return a->getId() < b->getId(); });
🌟 Важно: Стандартные алгоритмы C++ (std::find, std::sort, std::copy) работают не с контейнерами, а с итераторами. Это позволяет использовать один и тот же алгоритм для любых контейнеров: вектора, списка, массива, собственной коллекции.

🎯 Прикладная задача

Условие:

В IT-компании работают разработчики, менеджеры и тестировщики. Необходимо:

  1. Вывести всех сотрудников с эффективностью выше 85%
  2. Отсортировать сотрудников по зарплате и по ID
  3. Найти сотрудника по заданному ID

Решение:

// Задача 1: Вывод сотрудников с эффективностью > 85%
void demonstrateHighEfficiencyEmployees(const MyContainer<Employee*>& container) {
    std::cout << "\n=== High Efficiency Employees (Efficiency > 85%) ===" << std::endl;
    
    for (auto it = container.begin(); it != container.end(); ++it) {
        double efficiency = (*it)->getEfficiency();
        if (efficiency > 0.85) {
            (*it)->displayInfo();
            std::cout << " - Efficiency: " << (efficiency * 100) << "%" << std::endl;
        }
    }
}

📊 API класса MyContainer<T>

Метод Описание Сложность
MyContainer() Конструктор по умолчанию O(1)
~MyContainer() Деструктор (освобождает память) O(n)
push_back(const T&) Добавляет элемент в конец O(1) амортизированно
pop_back() Удаляет последний элемент O(1)
operator[](size_t) Доступ по индексу O(1)
size() Возвращает количество элементов O(1)
empty() Проверка на пустоту O(1)
clear() Очищает коллекцию O(1)
begin() / end() Возвращают итераторы O(1)
