# Семинар 5: Создание коллекции. Шаблоны классов и итераторы

## Предметная область: IT-компания

Проект демонстрирует реализацию шаблонного класса-коллекции `MyContainer<T>` с внутренним итератором на примере предметной области IT-компания (классы `Developer`, `Manager`, `Tester`, наследующие от абстрактного класса `Employee`).

---

└── README.md # Документация
```

## Реализованный функционал

### 1. Шаблонный класс MyContainer<T>

Класс-коллекция, представляющий динамический массив, который может хранить объекты любого типа.

**Основные методы:**
- `push_back(const T& value)` – добавление элемента в конец
- `pop_back()` – удаление последнего элемента
- `operator[](size_t index)` – доступ по индексу
- `size()` – получение количества элементов
- `empty()` – проверка на пустоту
- `clear()` – очистка коллекции

### 2. Внутренний итератор

Реализован класс `Iterator` с поддержкой категории `random_access_iterator_tag`.

**Возможности итератора:**
- Разыменование (`operator*`, `operator->`)
- Инкремент/декремент (`++`, `--`)
- Арифметические операции (`+`, `-`, `+=`, `-=`)
- Сравнение (`==`, `!=`, `<`, `>`, `<=`, `>=`)
- Индексирование (`operator[]`)

### 3. Использование std::find и std::sort

В программе демонстрируется применение стандартных алгоритмов к собственной коллекции:

- `std::find_if` – поиск сотрудника по ID
- `std::sort` – сортировка сотрудников по зарплате и по ID

### 4. Прикладная задача

Решена задача поиска и отображения сотрудников с высокой эффективностью (более 85%).

## Сборка и запуск

### Требования

- CMake 3.10 или выше
- Компилятор с поддержкой C++17 (GCC, Clang, MSVC)

### Инструкция

```bash
# Переход в директорию проекта
cd sem5_PCS

# Очистка предыдущей сборки (опционально)
rm -rf build

# Создание директории сборки
mkdir build && cd build

# Генерация файлов сборки
cmake ..

# Компиляция
make

# Запуск программы
./it_company_collection
